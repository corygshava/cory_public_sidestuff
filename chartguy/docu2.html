Getting Started

ChartForge is designed to be easy to use with minimal setup. This guide will help you get started in just a few minutes.

Installation

You can install ChartForge via npm or include it directly from a CDN.

Terminal 
npm install chartforge
HTML 
<script src="https://cdn.jsdelivr.net/npm/chartforge@1.0.0/dist/chartforge.min.js"></script>
Terminal 
yarn add chartforge

Basic Usage

Once installed, you can create your first chart with just a few lines of code:

JavaScript 
import ChartForge from "chartforge";

const chart = new ChartForge("#chart-container", {
    type: "bar",
    data: {
        labels: ["Jan", "Feb", "Mar", "Apr", "May"],
        values: [12, 19, 3, 5, 2]
    },
    options: {
        responsive: true,
        title: "Monthly Sales"
    }
});

Configuration

ChartForge uses a simple configuration object to customize your charts. Below are the common options available for all chart types.

Property Type Default Description
responsive Boolean true Whether the chart should resize when its container resizes
width Number null Explicit width for the chart (in pixels)
height Number null Explicit height for the chart (in pixels)
title String null Chart title displayed above the chart
colors Array Pre-defined palette Array of color strings for the chart elements
animation Boolean true Whether to animate the chart on load

Bar Chart

Bar charts are excellent for displaying categorical data. They can be oriented vertically or horizontally.

Basic Example

JavaScript 
const barChart = new ChartForge("#bar-chart", {
    type: "bar",
    data: {
        labels: ["Red", "Blue", "Yellow", "Green", "Purple", "Orange"],
        values: [12, 19, 3, 5, 2, 3]
    },
    options: {
        responsive: true,
        title: "Favorite Colors Survey",
        colors: ["#FF6384", "#36A2EB", "#FFCE56", "#4BC0C0", "#9966FF", "#FF9F40"]
    }
});

Bar Chart Specific Options

Property Type Default Description
orientation String "vertical" Bar orientation: "vertical" or "horizontal"
barThickness Number auto Thickness of bars in pixels
barPercentage Number 0.8 Percentage of available width to use for bars (0-1)
categoryPercentage Number 0.8 Percentage of available width to use for each category (0-1)

Line Chart

Line charts are perfect for showing trends and changes over time or continuous data.

Basic Example

JavaScript 
const lineChart = new ChartForge("#line-chart", {
    type: "line",
    data: {
        labels: ["Jan", "Feb", "Mar", "Apr", "May", "Jun"],
        values: [65, 59, 80, 81, 56, 55]
    },
    options: {
        responsive: true,
        title: "Monthly Revenue",
        tension: 0.4,
        fill: false
    }
});

Line Chart Specific Options

Property Type Default Description
tension Number 0 Bezier curve tension of the line (0 for no curves)
fill Boolean false Whether to fill the area under the line
pointRadius Number 3 Radius of the data points in pixels
pointHoverRadius Number 5 Radius of the data points when hovered (in pixels)

Scatter Chart

Scatter charts display the relationship between two variables using dots at the intersection of x and y values.

Basic Example

JavaScript 
const scatterChart = new ChartForge("#scatter-chart", {
    type: "scatter",
    data: {
        datasets: [{
            label: "Height vs Weight",
            data: [
                { x: 65, y: 75 },
                { x: 59, y: 80 },
                { x: 80, y: 56 },
                { x: 81, y: 55 },
                { x: 56, y: 40 },
                { x: 55, y: 35 }
            ]
        }]
    },
    options: {
        responsive: true,
        title: "Height vs Weight Correlation",
        pointRadius: 5
    }
});

Scatter Chart Specific Options

Property Type Default Description
pointRadius Number 3 Radius of the points in pixels
pointHoverRadius Number 5 Radius of the points when hovered (in pixels)
showLine Boolean false Whether to show a line connecting the points
tension Number 0 Bezier curve tension of the line (if showLine is true)

Styling

ChartForge provides flexible styling options to match your design requirements.

Custom Colors

You can specify custom colors for your charts:

JavaScript 
const chart = new ChartForge("#chart", {
    type: "bar",
    data: {
        labels: ["A", "B", "C"],
        values: [10, 20, 15]
    },
    options: {
        colors: ["#FF6384", "#36A2EB", "#FFCE56"]
    }
});

Global Styles

You can also set global styles that apply to all charts:

JavaScript 
ChartForge.defaults.font.family = "Arial";
ChartForge.defaults.font.size = 14;
ChartForge.defaults.color = "#333";

Events

ChartForge emits events that you can listen to for interactive functionality.

Event Description Parameters
click Fired when clicking on the chart event, elements
hover Fired when hovering over chart elements event, elements
resize Fired when the chart is resized newSize

Example: Handling Click Events

JavaScript 
const chart = new ChartForge("#chart", {
    type: "bar",
    data: {
        labels: ["A", "B", "C"],
        values: [10, 20, 15]
    }
});

chart.on("click", (event, elements) => {
    if (elements.length > 0) {
        const index = elements[0].index;
        const value = chart.data.values[index];
        alert(`Clicked on value: ${value}`);
    }
});

API Reference

The ChartForge API provides methods for programmatically interacting with charts.

Chart Instance Methods

Method Description Parameters
update() Updates the chart with new data or options config (optional)
resize() Manually triggers a resize of the chart none
destroy() Destroys the chart instance and cleans up resources none
on(event, handler) Adds an event listener to the chart eventName, callback
off(event, handler) Removes an event listener from the chart eventName, callback

Example: Updating Chart Data

JavaScript 
const chart = new ChartForge("#chart", {
    type: "line",
    data: {
        labels: ["Jan", "Feb", "Mar"],
        values: [10, 20, 15]
    }
});

// Update with new data
chart.update({
    data: {
        labels: ["Jan", "Feb", "Mar", "Apr"],
        values: [12, 19, 3, 5]
    }
});
